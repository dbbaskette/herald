### Task 2: In-Memory ChatMemory Fallback (#223)

**Files:**
- Modify: `herald-bot/src/main/java/com/herald/agent/HeraldAgentConfig.java`

Add a fallback bean method:
```java
@Bean
@ConditionalOnMissingBean(ChatMemoryRepository.class)
public ChatMemoryRepository inMemoryChatMemoryRepository() {
    return new InMemoryChatMemoryRepository();
}
```

This means: when no `herald.memory.db-path` is set, DataSourceConfig skips, no JsonChatMemoryRepository is created, so this fallback kicks in. The existing `chatMemory()` bean (gated on ChatMemoryRepository presence) will then wrap it in MessageWindowChatMemory.

**IMPORTANT:** Also remove `@ConditionalOnBean(ChatMemoryRepository.class)` from the `chatMemory()` bean — it's no longer needed since we always have a ChatMemoryRepository (either JDBC or in-memory). Or keep it but ensure the fallback is created first.

Actually — keep `@ConditionalOnBean` on chatMemory(). The fallback bean provides the ChatMemoryRepository, so the condition is satisfied. This is cleaner.

Commit: `feat: add in-memory ChatMemory fallback for ephemeral mode (#223)`
