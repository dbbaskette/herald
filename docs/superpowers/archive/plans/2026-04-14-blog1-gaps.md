# Blog 1 Gaps — Classpath Skills + Anthropic Native Skills

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the two remaining Blog 1 gaps: (1) classpath/JAR skill packaging so skills can be bundled inside the application JAR, and (2) Anthropic Native Skills API integration so Claude can generate XLSX/DOCX/PPTX/PDF documents.

**Architecture:** Classpath skills use the existing `ReloadableSkillsTool(directory, resources)` constructor — we just need to wire `Resource[]` from a config property. Anthropic skills are set on `AnthropicChatOptions` at `ChatClient` build time — they flow through to the API automatically.

**Tech Stack:** Spring `ResourcePatternResolver`, `AnthropicChatOptions.builder().skill()`, existing `ReloadableSkillsTool`

```java
@Test
void loadsSkillFromClasspathResource() throws IOException {
    // Create a temporary skill file to use as a "classpath" resource
    Path skillDir = tempDir.resolve("classpath-skill");
    Files.createDirectories(skillDir);
    Files.writeString(skillDir.resolve("SKILL.md"), """
            ---
            name: classpath-skill
            description: A skill loaded from classpath
            ---
            Classpath skill content.
            """);

    var resource = new org.springframework.core.io.FileSystemResource(
            skillDir.resolve("SKILL.md").toFile());
    var tool = new ReloadableSkillsTool(
            tempDir.resolve("nonexistent-dir").toString(),
            List.of(resource));
    assertThat(tool.getSkills()).hasSize(1);
    assertThat(tool.getSkills().get(0).name()).isEqualTo("classpath-skill");
    assertThat(tool.hasSkills()).isTrue();
}

@Test
void combinesDirectoryAndClasspathSkills() throws IOException {
    // Directory skill
    Path dirSkill = tempDir.resolve("dir-skill");
    Files.createDirectories(dirSkill);
    Files.writeString(dirSkill.resolve("SKILL.md"), """
            ---
            name: dir-skill
            description: From directory
            ---
            Directory content.
            """);

    // Classpath skill
    Path cpSkill = tempDir.resolve("cp-skill");
    Files.createDirectories(cpSkill);
    Files.writeString(cpSkill.resolve("SKILL.md"), """
            ---
            name: cp-skill
            description: From classpath
            ---
            Classpath content.
            """);

    var resource = new org.springframework.core.io.FileSystemResource(
            cpSkill.resolve("SKILL.md").toFile());
    var tool = new ReloadableSkillsTool(tempDir.toString(), List.of(resource));
    assertThat(tool.getSkills()).hasSize(2);
    assertThat(tool.getSkills()).extracting(s -> s.name())
            .containsExactlyInAnyOrder("dir-skill", "cp-skill");
}
```

- [ ] **Step 3: Wire classpath resources in HeraldAgentConfig**

In `HeraldAgentConfig.java`, update the `reloadableSkillsTool` bean to resolve classpath skill resources:

```java
@Bean
public ReloadableSkillsTool reloadableSkillsTool(
        @Value("${herald.agent.skills-directory:skills}") String skillsDirectory,
        ResourcePatternResolver resourceResolver) throws IOException {
    // Resolve classpath skill resources (e.g., classpath:skills/*/SKILL.md)
    Resource[] classpathSkills = resourceResolver.getResources("classpath:skills/*/SKILL.md");
    return new ReloadableSkillsTool(skillsDirectory, List.of(classpathSkills));
}
```

Add the import:
```java
import org.springframework.core.io.support.ResourcePatternResolver;
```
